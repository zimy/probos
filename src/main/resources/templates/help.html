Таблица на странице, переадресовывающей сюда, содержит информацию о результатах симуляции алгоритма управления кешем LRU. Принцип LRU описывает такое поведение кеша, при котором наиболее приоритетной к вытеснению является страница, к которой не обращались дольше всего. Для симуляции используется собственная реализация кеша LRU, содержащая целые числа в качестве полезной нагрузки. Для полезных применений может быть легко модифицирована для использования с другими типами данных. Настройка приложения производится при помощи трёх GET-параметров: cycles, cache, pages.

  • Cache отвечает за размер кеша
  • Pages отвечает за количество страниц, доступных для выбора
  • Cycles определяет количество экспериментов

Параметры указываются в URI запроса к странице /os с использованием стандартного синтаксиса подстановки GET-параметров, например так: domain/os/?cache=6&pages=32&cycles=50.

Страница для помещения в кеш является значением дискретной равномерно распределённой случайной величины, использующей вихрь Мерсенна в качестве генератора входных данных.

Для сборки используется система непрерывной интеграции Travis CI, система сборки Gradle. После успешной сборки выполняется запуск проекта в облачном инстансе RedHat OpenShift.
